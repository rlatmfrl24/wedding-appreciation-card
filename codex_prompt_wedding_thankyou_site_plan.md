# Codex Prompt — 결혼 감사장 웹사이트 제작 플랜

아래 전체 내용을 그대로 Codex에 붙여넣어 주세요.

---

너는 **감성적 브랜드 경험 설계에 강한 시니어 UX/UI 디자이너이자 React 프론트엔드 아키텍트**다.  
나는 **결혼식 이후 하객, 친지, 가족, 그리고 멀리서 축하해주신 분들께 감사의 마음을 전하는 모바일 중심 감사장 웹사이트**를 만들고자 한다.

이 웹사이트의 목적은 단순히 정보를 보여주는 것이 아니라, 방문자가 페이지에 들어왔을 때 **따뜻함, 진심, 여운, 품격, 잔잔한 감동**을 느끼게 하는 것이다.  
형식은 격식을 갖추되 차갑지 않아야 하며, 전체 분위기는 **감성적이고 부드럽고 세련된 인상**이어야 한다.

이 프로젝트는 **React 프로젝트로 제작 및 배포**할 예정이며, **결혼식 직후부터 신혼여행 출발 사이에 빠르게 배포 가능한 수준의 MVP와 확장 가능한 구조**가 모두 필요하다.

## 프로젝트 기본 정보

- 신랑/신부 이름: **김슬기 / 윤민지**
- 사이트에 넣을 대표 사진: **있음**
- 사진 스타일: **스튜디오 촬영 사진**
- 배포 목표 시점: **결혼식 직후부터 신혼여행 출발 사이**
- 주요 접속 환경: **모바일 우선**
- 핵심 목표: **감사 메시지를 품위 있게 전달하고, 링크 공유만으로 쉽게 볼 수 있는 웹사이트 제작**

## 중요한 전제

- 배포 시점이 촉박하므로, **짧은 시간 안에 완성 가능한 구조와 우선순위가 분명한 MVP 설계**가 중요하다.
- 동시에 추후에 다음 기능을 자연스럽게 확장할 수 있어야 한다.
  - 사진 갤러리
  - 짧은 영상
  - 부모님/가족 감사 문구
  - 연락처 또는 짧은 인사말
  - 방명록 또는 축하 메시지
- 너무 많은 기능보다 **감사의 마음이 자연스럽게 전달되는 흐름**이 우선이다.
- 상업적인 웨딩 사이트처럼 과장되거나 화려한 느낌은 피하고, **“감사 편지를 디지털로 정갈하게 옮긴 경험”**을 목표로 한다.

## 원하는 무드

다음 감정을 핵심 톤으로 삼아라.

- 따뜻함
- 부드러움
- 은은함
- 진심
- 여백의 미
- 단정함
- 잔잔한 감동
- 오래 남는 여운

다음과 같은 정서를 참고할 수 있다.

- 종이 감사장
- 크림톤 종이
- 한지 질감
- 햇살이 비치는 듯한 밝고 은은한 분위기
- 작은 꽃 또는 식물 모티프
- 필름 사진 같은 포근한 정서
- 절제된 손글씨 포인트

단, 아래 요소는 과하게 사용하지 말고 **매우 절제된 방식**으로 사용해야 한다.

- 꽃 장식
- 필름 프레임
- 빛 번짐
- 손글씨 폰트
- 애니메이션

## 감사 문구

아래 문구를 웹사이트의 핵심 카피로 활용할 예정이다.

소중한 분들께,

저희 두 사람이 함께 걸어갈 새로운 시작의 날에  
바쁘신 가운데 귀한 걸음으로 자리를 빛내 주시고,  
따뜻한 축복과 마음을 전해 주신 모든 분들께 진심으로 감사드립니다.

결혼식 당일, 보내주신 환한 미소와 정성 어린 축하 덕분에  
저희는 더없이 행복하고 벅찬 마음으로 그날을 맞이할 수 있었습니다.  
직접 함께해 주신 하객 여러분과 친지분들,  
늘 곁에서 아낌없는 사랑을 베풀어 주신 가족들,  
그리고 멀리서도 깊은 마음으로 축복해 주신 모든 분들의 정성을  
저희는 오래도록 소중히 간직하겠습니다.

한 분 한 분의 축복은 저희에게 큰 기쁨이자  
앞으로의 삶을 살아가는 데 든든한 힘이 되었습니다.  
보내주신 따뜻한 마음에 보답할 수 있도록  
서로 아끼고 존중하며, 감사한 마음을 잊지 않고  
아름답고 단단한 가정을 이루어 가겠습니다.

귀한 마음 보내주신 모든 분들께 다시 한 번 깊이 감사드리며,  
언제나 건강과 평안, 그리고 행복이 함께하시기를 진심으로 기원합니다.

감사합니다.

**김슬기 · 윤민지 올림**

---

위 내용을 바탕으로, **“따뜻하고 감성적인 결혼 감사장 웹사이트” 제작 플랜**을 매우 구체적이고 실무적으로 작성해줘.

반드시 아래 조건을 반영해줘.

## 반드시 반영할 조건

### 1) 프로젝트 목적
- 결혼식 후 감사의 마음을 온라인으로 정중하게 전달
- 모바일로 접속하는 사용자가 많을 것을 고려
- 과하지 않고 정갈하며 감성적인 분위기
- 가족과 어른들도 편하게 볼 수 있도록 가독성과 사용성을 중요하게 설계
- 공유하기 쉽고 배포가 간단해야 함
- **시간 제약이 있으므로 빠르게 구현 가능한 MVP 범위를 먼저 정의**해야 함

### 2) 원하는 분위기
- 따뜻함, 부드러움, 은은함, 진심, 여백의 미
- 지나치게 화려하거나 트렌디한 웨딩 상업 사이트 느낌은 피하기
- “감사 편지”를 디지털 공간으로 옮긴 듯한 감성
- 스튜디오 사진이 들어가더라도 차갑거나 광고 비주얼처럼 보이지 않게 톤을 조절할 것
- 애니메이션은 최소한으로, 느리고 부드럽고 절제된 방식 선호

### 3) 기술 전제
- React 프로젝트로 제작 예정
- 빠른 배포가 중요하므로 구현 난이도와 배포 편의성을 같이 고려
- 유지보수와 확장성을 고려한 구조 필요
- 추후 사진 갤러리, 짧은 영상, 연락처/인사말, 방명록 또는 축하메시지 섹션을 추가할 가능성 있음
- **정적 배포에 유리한 구조인지도 함께 판단**할 것

### 4) 결과물의 성격
답변은 추상적인 감성 표현만 나열하지 말고,  
실제로 **디자인 시안과 React 프로젝트 구조를 바로 잡을 수 있을 정도로 구체적이고 실무적**으로 작성해줘.

또한 아래 원칙을 꼭 지켜줘.

- “예쁜 웨딩 사이트”가 아니라 **“감사의 마음이 전해지는 디지털 감사장”**이어야 한다.
- 감성은 살리되 **사용성, 가독성, 속도, 배포 편의성**도 함께 고려한다.
- 너무 많은 기능보다 **메시지 전달력**이 우선이다.
- 한국어 콘텐츠 중심의 모바일 웹사이트로 생각한다.
- 배포 목표가 촉박하므로 **즉시 배포 가능한 MVP**와 **추가 확장안**을 구분해서 제안한다.

## 아래 순서로 답변해줘

### A. 프로젝트 한 줄 정의
이 웹사이트의 콘셉트를 한 줄로 정의

### B. 핵심 감성 키워드 5~8개
브랜드/디자인 무드 키워드 제안

### C. 전체 UX 콘셉트
- 사용자가 처음 방문해서 마지막까지 어떤 감정 흐름을 경험하는지
- 첫 화면 인상, 스크롤 흐름, 마지막 여운까지 설명
- 감사 메시지가 자연스럽게 읽히도록 어떤 호흡으로 콘텐츠를 배치할지 설명

### D. 정보 구조(IA)
- 이 사이트에 들어가면 좋을 섹션들을 우선순위 순으로 제안
- 각 섹션의 목적과 담길 콘텐츠 설명
- **MVP에 꼭 필요한 섹션**과 **추후 확장 가능한 섹션**을 구분

### E. 와이어프레임 수준의 페이지 구성
모바일 우선으로, 각 섹션별 화면 구성과 레이아웃을 구체적으로 설명  
예:
- 상단 히어로 영역
- 짧은 인트로 카피
- 대표 사진 영역
- 본문 메시지 블록
- 사진 블록
- 엔딩 메시지
- 공유 유도 영역

각 섹션별로 다음을 함께 설명해줘.
- 콘텐츠 우선순위
- 화면 내 배치 방식
- 시선 흐름
- 스크롤 길이와 밀도
- 어르신도 읽기 쉬운지 여부

### F. 비주얼 디자인 가이드
아래 항목을 포함해서 제안

- 컬러 팔레트(메인/서브/배경/텍스트/포인트)
- 타이포그래피 방향(국문 웹폰트 분위기 포함)
- 제목/본문/인용문/버튼의 서체 위계
- 여백, 라운드, 선, 그림자, 배경 텍스처 사용 방향
- 사진/일러스트 스타일 제안
- 스튜디오 사진을 너무 딱딱하거나 상업적으로 보이지 않게 연출하는 UI 방법
- 꽃, 종이, 필름, 빛 번짐, 손글씨 느낌 등을 어떤 방식으로 절제해 활용할지 설명

### G. UI 컴포넌트 제안
React에서 재사용 가능한 컴포넌트 중심으로 정리
예:
- HeroSection
- IntroNote
- MessageSection
- QuoteBlock
- PhotoHighlight
- PhotoGallery
- FamilyThanksSection
- ClosingNote
- FloatingShareButton
- FooterNote

각 컴포넌트별로 아래를 정리해줘.
- 역할
- 포함할 콘텐츠
- 필요한 props 예시
- 스타일 방향
- MVP 포함 여부

### H. 인터랙션 및 모션 가이드
- 과하지 않은 애니메이션 원칙
- 페이드 인, 슬로우 리빌, 스크롤 트리거 사용 여부
- 패럴랙스 필요 여부
- 모바일 성능을 해치지 않으면서 감성을 살리는 방법
- 접근성을 해치지 않는 모션 기준
- prefers-reduced-motion 대응 여부

### I. 접근성/사용성 체크포인트
- 어르신도 보기 쉬운 폰트 크기
- 명도 대비
- 버튼 크기와 터치 영역
- 모바일 중심 UX
- 읽기 흐름과 스크롤 피로도
- 이미지 로딩 최적화
- 배경음악 사용 여부 판단
- 카카오톡 등 메신저로 공유했을 때 첫인상까지 고려

### J. React 구현 관점의 추천 구조
다음을 포함해서 제안

- React 프로젝트 추천 방식 (예: Vite 기반 React / Next.js 등)
- 왜 그 방식을 추천하는지
- 폴더 구조 예시
- 컴포넌트 구조
- 스타일링 방식 추천 (예: CSS Modules / styled-components / Tailwind 중 성격에 맞는 선택과 이유)
- 정적 자산 관리 방식
- 대표 사진/갤러리 사진 처리 방식
- 웹폰트 적용 방식
- 메타태그/SEO/OG 이미지/카카오톡 공유 대응 포인트
- 배포 방식 추천 (예: Vercel, Netlify 등)과 이유

### K. 단계별 제작 플랜
아래 단계로 나눠서 작성
1. 리서치/무드보드  
2. IA 및 카피 정리  
3. 와이어프레임  
4. 비주얼 디자인 시안  
5. React 컴포넌트 설계  
6. 구현  
7. QA  
8. 배포  
9. 공유/운영  

각 단계별로 다음을 정리해줘.
- 해야 할 일
- 산출물
- 체크리스트
- 주의사항
- 빠르게 진행해야 할 때 생략 가능한 것과 절대 생략하면 안 되는 것

### L. 추천 무드보드 방향 3가지
서로 결이 다른 방향으로 제안해줘. 예:
- 편지 같은 클래식 감성
- 햇살과 필름 사진 같은 내추럴 감성
- 미니멀하고 정갈한 프리미엄 감성

각 방향마다 다음을 설명해줘.
- 핵심 분위기
- 컬러 방향
- 폰트 방향
- 레이아웃 특징
- 사진 활용 방식
- 어떤 사용자에게 더 편안하게 느껴질지
- 이번 프로젝트와의 적합도

### M. 최종 추천안
위 3가지 중 이번 감사장 사이트에 가장 잘 어울리는 방향 1개를 골라  
그 이유와 함께 추천해줘.

반드시 아래 관점으로 판단해줘.
- 감사 메시지 전달력
- 모바일 사용성
- 배포 일정의 현실성
- 스튜디오 대표 사진과의 조화
- 어른들이 봐도 편안한 톤

### N. MVP 범위와 확장 로드맵
반드시 아래 두 가지를 분리해서 제안해줘.

#### 1) 결혼식 직후 바로 배포 가능한 MVP
- 꼭 필요한 화면/섹션
- 최소한의 컴포넌트
- 필요한 이미지 수
- 구현 난이도
- 빠른 QA 포인트

#### 2) 이후 여유가 생기면 추가할 확장안
- 사진 갤러리 확대
- 영상 추가
- 부모님 감사 문구 섹션
- 방명록/메시지 기능
- 공유 기능 고도화

### O. 실무용 체크리스트
디자이너와 개발자가 바로 착수할 수 있도록  
필수 준비물, 결정해야 할 항목, 빠뜨리기 쉬운 요소를 체크리스트로 정리해줘.

### P. 바로 화면 설계에 들어갈 수 있는 핵심 요약
마지막에는 10줄 내외로,  
이 기획을 바탕으로 바로 화면 설계에 들어갈 수 있도록 핵심만 압축해서 정리해줘.
